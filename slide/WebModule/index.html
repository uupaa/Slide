WebModule

Web Application Module

@uupaa - 2014-04-01

WebModule は

シンプルで
テストがあり
どこでも動作し
品質が数値化され
良質のドキュメントがある
そんな部品群を量産するための
開発フローと関連ツールを提供します。

こんな方にお勧めです

WebModule のレールに乗ると、
あらかじめ用意されているモジュールの雛形と
ツールの使い方の流れを学ぶことで、
JavaScriptを使ったモジュールを開発するための
最初の一歩を踏み出すことができます。

特にお勧めしたい方々

「どこでも動くJSの部品を作りたいけど、
どこから手を付ければ、
どうすればいいのか分からない」な、
まだ自分なりの型を持っていない人には
特にお勧めです(新人さんとか)。

モザイクなフレームワーク

WebModule は、
Browser + npm + GitHub + Markdown + ClosureCompiler + JSHint + Plato + Help.js を組み合わせて開発を進めます。

こんなイメージ

運用をシンプルにするための
決まり事(型)があります

  • 公開可能な ソースコードは GitHub と npm に置きます
    • 公開できないものは、今までどおり別の場所で管理します
  • ドキュメントは GitHub/wiki に書きます
  • ドキュメントの書式は Markdown です

ソースコードとドキュメントで
履歴を別々に管理します。

  • ソースコードの履歴は git に任せます
  • ドキュメントの履歴は GitHub/wiki の機能を利用します

これまでのデファクトを破ります

WebModule は、Help.js による
ソースコードとドキュメントの
疎結合を推奨しています。


JavaDoc / JsDoc は使いません。
(併用は可能です)

これまでの定型や定説に異を唱えるチャレンジになります。

コードを健全に保つ仕組みを提供します


ソースコードの静的解析を行い、
品質を点数化し、結果をグラフで表示します。


酷い修正を行うとグラフも酷い事になります。

どこでも動く コードになります


6つの環境で一斉にテストします。

  1. Node.js + SourceCode
  2. Node.js + MinifiedCode
  3. Browser + SourceCode
  4. Browser + MinifiedCode
  5. WebWorker + SourceCode
  6. WebWorker + MinifiedCode

WebModule の使い方

WebModule をインストールします

$ git clone git@github.com:uupaa/WebModule.git
$ cd WebModule
$ npm install
$ cd ..

MyWebModule を作成します

$ git clone git@github.com:GITHUB-USER-NAME/MyWebModule.git
$ cd MyWebModule
$ node ../WebModule/clone GITHUB-USER-NAME
$ npm install

以下のコマンドで開発を進めます

$ edit ../lib/MyModule.js

$ npm run watch             <- lib/MyModule.js を監視して自動ビルド
$ npm run build             <- release/MyModule.min.js をビルド
$ npm run test              <- lib/MyModule.js と release/MyModule.min.js をテスト
$ npm run lint              <- lib/MyModule.js を評価

Browser, WebWorkers, Node.js で使う

<!-- DependencyWebModules -->
<script src="../node_modules/module1.js/lib/WebModule1.js"></script>
    :
<script src="../lib/MyWebModule.js"></script>
// DependencyWebModules
importScripts("../node_modules/module1.js/lib/WebModule1.js");
    :
importScripts("../lib/MyWebModule.js");
// DependencyWebModules
require("../node_modules/module1.js/lib/WebModule1.js");
    :
require("../lib/MyWebModule.js");

パスを解決できる環境(Node.js)では、
モジュールのパスを短く記述できます。

require("module1.js");
    :
require("../lib/MyWebModule.js");
